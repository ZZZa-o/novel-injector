## 2026-05-13

1. 建立插件第一套完整工作流：上传 `.txt` 小说、识别文本编码、按设定 KB 大小切分原文，并显示文件大小、分段数量和处理进度。
2. 增加清洗 API 配置，包括接口地址、API Key、模型、模型列表拉取、流式输出和基础错误提示。
3. 建立 API 请求限流与并发信号量，减少连续清洗长篇小说时触发服务端频率限制的概率。
4. 完成 AI 清洗压缩流程：每段输出压缩正文，并解析结构化 `ni_meta` 数据。
5. 首次形成剧情节点体系，区分主线 `main`、支线 `sub` 和关键转折 `pivot`，记录标题、正文、时间和地点。
6. 首次形成角色数据体系，提取角色名称、分类、行为摘要和性格依据，并提供手动新增、删除与 AI 生成入口。
7. 增加剧情的时间轴/分类视图，以及按关键转折自动划分阶段、手动新建阶段、分配未归属节点等功能。
8. 增加按阶段选择的 Embedding 向量化、阶段独立索引和基础向量召回注入设置。
9. 增加注入深度、最多召回片段数、相似度阈值等基础参数。

## 2026-05-14

2. 增加单次 API 请求超时时间设置，让长文本模型无响应时可自动中止并重试。

5. 角色人设由简单简介扩展为身份、外貌、性格、关系四类字段，并支持逐字段编辑。

7. 增加向量阶段选择面板，可全选、全不选或仅选择尚未向量化的阶段。
8. 新增“当前偏差”分析入口，用当前对话与原著阶段资料对比剧情、人物行为和支线偏离情况。
9. 细化清洗提示词中的主线、支线和关键转折判定标准，限制滥用 `pivot`，并要求记录支线说明和伏笔关联。

## 2026-05-16

## 2026-05-20

4. 增加压缩正文最低长度和 10:1～15:1 压缩比约束，避免模型过度缩写导致向量检索缺少细节。

1. 优化向量批量大小选择器，将控件宽度压缩并居中显示，减少配置行占用。
