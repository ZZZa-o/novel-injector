# Novel Injector — 小说上下文注入插件

将长篇小说的原著内容动态注入 SillyTavern 对话上下文，让 AI 在续写时始终贴合原著剧情、角色和时间线。

---

## 功能概览

```
上传小说 .txt
    ↓
分段 → AI 清洗压缩 → 提取剧情节点 + 角色人设
    ↓
（可选）提取世界设定 / 分析文风
    ↓
手动划分阶段（或按转折点自动划分）
    ↓
（可选）对指定阶段向量化
    ↓
发送消息时自动注入：
  · 已向量阶段  → 语义召回最相关片段
  · 未向量阶段  → 直接注入该阶段剧情节点（或压缩原文）
  · 角色人设    → 注入已启用角色的身份/外貌/性格/关系
  · 世界设定    → 注入世界边界/特殊机制/社会规则等大类
  · 文风指南    → 注入 AI 生成的写作执行规则
  · 全局提示词  → 每次 AI 调用均前置注入
    ↓
（可选）开启穿书模式：
  状态栏/弹窗显示节点进度 → 手动勾选完成 → 自动注入推进提示词
  → AI 自由推演三条后续走向
```

---

## 安装

将 `novel-injector` 文件夹放入 SillyTavern 的 `public/extensions/third-party/` 目录，重启或刷新页面后点击顶栏书本图标（📖）即可打开插件面板。

---

## 界面结构

底部导航栏分为五个页面：**处理 → 剧情 → 角色 → 阶段 → 设置**。

---

## 处理页

### 上传小说

- 点击上传区或拖拽 `.txt` 文件（支持 UTF-8 / UTF-16 / GBK / GB18030 自动检测）
- 上传后按设定的分段大小（默认 100 KB）自动切分，每段显示状态（待处理 / 处理中 / 已完成 / 失败）
- 可在上传后随时调整分段大小，列表实时更新

---

### 清洗压缩

将原始小说文本压缩并提取结构化元数据，每段压缩比约 10:1～15:1。

清洗 AI 的角色设定为**资深文学编辑**，长期处理各类题材投稿，对敏感情节（亲密关系、暴力、古典文学中的特殊描写）均以客观文学视角处理，记录具体行为事实及剧情意义，不回避、不模糊。

**API 设置（点击齿轮图标展开）：**

| 字段 | 说明 |
|---|---|
| API 端点 | 兼容 OpenAI 格式的 chat completions 接口 |
| API Key | 对应密钥 |
| 模型 | 手动输入或点击刷新按钮自动拉取模型列表 |
| 流式输出 | 开启后以 SSE 流式接收响应，可加快首字节时间 |
| 每分钟限 N 次 | 全局 API 限速，超出后请求自动排队等待（0 = 不限） |
| 无响应超过 N 分钟 | 单段超时后自动中止并重试，最多重试 3 次 |

**提示词（点击代码图标展开）：**  
可自定义压缩提示词，点击还原按钮恢复内置默认提示词。

**清洗操作：**

- **开始全自动清洗** — 顺序处理所有待处理段，已完成的段自动跳过（支持断点续跑）
- **暂停** — 中止当前段请求，下次点击开始时从断点继续
- **跳过本段** — 放弃当前段，继续处理下一段
- **重试失败分段** — 批量重跑所有标记为失败的段
- **生成此段**（段落列表中） — 单独清洗某一段，不影响其他段

清洗完成后自动生成剧情节点和角色人设，并在剧情页、角色页展示。

> **注意：** 若 AI 返回的响应缺少结构化元数据（`ni_meta`），该段会自动重试（最多 3 次），重试前会尝试从响应正文中抢救裸 JSON。单独生成某段时若仍失败，状态将标记为失败并提示重试。

---

### 文风分析

对原著写作风格进行提炼，生成可直接指导 AI 续写的执行规则，注入时作为文风指南使用。

**两种采样模式（点击齿轮图标切换）：**

| 模式 | 说明 |
|---|---|
| 采样模式 | 从已上传的某一原始段落中截取前 N 字作为分析样本（需在当前会话中上传过文件） |
| 范文模式 | 手动粘贴任意范文文本，适合未上传文件时使用 |

采样模式下可选择段落编号和截取字数（默认 1000 字）。

**提示词（点击代码图标展开）：**  
可自定义文风分析提示词，默认提示词要求 AI 以「执行指令」而非「风格评价」输出，每条规则必须落实到具体写法操作。内置维度涵盖：叙事节奏、对话风格、描写偏好、情感克制程度、结构习惯、人称惯例、用词风格。

生成结果可在文本框内直接编辑后保存，保存的文风数据随小说快照一并持久化。

---

### 向量化

对清洗后的压缩文本按阶段建立向量索引，存储于浏览器 IndexedDB，用于发送消息时的语义召回。

**向量 API 设置（点击齿轮图标展开）：**

| 字段 | 说明 |
|---|---|
| API 端点 | OpenAI 兼容的 Embedding 接口根路径（如 `https://api.openai.com/v1`） |
| Embedding API Key | 对应密钥 |
| 模型 | 推荐 `text-embedding-3-large` |

**阶段选择（点击列表图标展开）：**  
勾选要向量化的阶段，支持全选 / 全不选 / 仅未向量。点击「开始向量化」后仅处理勾选阶段，已向量的阶段数据会被同阶段新数据覆盖。

**🔍 诊断（阶段选择面板内）：**  
点击「诊断」按钮，读取 IndexedDB 实际存储数据并以弹窗输出报告，包含：

- 当前 `novelKey`
- IndexedDB 中的总向量块数
- 各阶段实际存储的向量块数明细
- 异常检测：若某阶段状态标记为「已向量」但 IndexedDB 中实际无对应数据，会标注 ⚠️ 并给出可能原因（API Key/地址/模型有误、限速截断等）和修复建议

召回效果不符合预期、或向量化状态与实际不一致时，优先用此功能排查。

**补全缺失（按需显示）：**  
插件加载时会异步对比 IndexedDB 已存向量块与应有的完整列表，若发现有遗漏块（如因 API 报错导致部分块未写入），则自动显示「补全缺失」按钮。点击后只补跑缺失的块，已存在的块不重复请求，完成后按钮自动隐藏。

**召回加权设置（向量 API 面板内）：**

| 字段 | 说明 | 默认 |
|---|---|---|
| 召回时提取的消息内容 | 填标签名后只提取该 XML 标签内的文字用于召回，多个匹配块全部拼接；留空则使用完整消息；若某条消息不含该标签，自动回退为完整消息 | 留空 |
| 取近几条对话 | 召回时向前取多少条消息参与语义匹配 | 3 |

召回时，取到的多条消息**不是拼成一段**，而是分别计算 embedding 后按时间远近加权合并（最新一条权重 1.0，每往前一条 × 0.5 指数衰减），再与向量库做相似度检索，让近期上下文对召回结果的影响更大。

---

### 注入设置

控制五类内容注入到 SillyTavern prompt 的位置和方式，点击卡片标题展开，内含五个手风琴面板：

| 面板 | 内容 | 默认位置 |
|---|---|---|
| 角色人设 | 已启用角色的身份/外貌/性格/关系 | 主提示前 |
| 阶段剧情（未向量） | 未向量化阶段的剧情节点或压缩原文 | 聊天内 |
| 向量块 | 语义召回片段 | 聊天内 |
| 世界设定 | 已启用大类的世界设定文本 | 主提示前 |
| 文风 | AI 生成的写作执行规则指南 | 主提示前 |

每个面板可独立设置**注入位置**（主提示前 / 主提示后 / 聊天内）、**注入深度**和**消息角色**（系统 / 用户 / 助手）。

向量块额外提供：
- **最多召回片段数**（TopK，默认 3）
- **向量相似度阈值**（默认 0.5，低于此值的片段被过滤）

## 剧情页

### 前文偏差分析

对比当前对话最近 10 条消息与已开启阶段的原著参考（已向量阶段走语义召回，未向量阶段取剧情节点文本），用 AI 评分四个维度：

- 主线剧情贴合度
- 人物行为贴合度
- 地点信息贴合度
- 支线完整度

结果以进度条百分比展示，附带总体分析摘要。

---

### 世界设定

从剧情节点中提取并凝练世界观框架，支持多个独立大类，各自可独立开关注入。

**内置大类：**

| 大类 | 说明 |
|---|---|
| 世界边界 | 这个世界存在什么、不存在什么（科技水平、特殊物质、超自然现象等） |
| 特殊机制 | 世界独有的规则、超自然机制及其限制（修炼/异能/系统体系等） |
| 社会规则 | 权力结构、社会阶层、法律与现实世界的差异（默认关闭） |

**操作：**

| 按钮 | 说明 |
|---|---|
| 开关图标 | 独立控制该大类是否注入 prompt |
| 重新生成 | 仅重新生成该大类的内容 |
| 编辑 | 展开可编辑文本域，手动修改后保存 |
| AI 全部生成 | 对所有大类调用 AI 提取（需先完成清洗） |
| ＋ 新增大类 | 手动添加自定义大类并填写 hint 说明 |

---

### 剧情节点管理

清洗后自动提取三类节点：**主线 / 支线 / 关键转折**。

**节点类型判定标准：**

- **主线（main）** — 直接推动故事核心矛盾发展，删去后主线逻辑链断裂
- **支线（sub）** — 有相对独立的起因经过结果，与核心矛盾无直接关联，丰富世界观/人物关系
- **关键转折（pivot）** — 同时满足：人物关系/立场不可逆改变、核心驱动力根本改变、对后续持续影响；大多数段落应为 0 个

**四个视图 Tab：**

- **时间轴** — 将主线 + 转折按 chunk 顺序合并展示，支线以分支链接形式挂靠在对应主线节点下，伏笔以书签标签展示
- **主线 / 支线 / 关键转折** — 独立列表，支持拖拽排序

**工具栏操作：**

| 按钮 | 说明 |
|---|---|
| 时序 | 时间轴 Tab 专属，按 `time` 字段重新排序，再次点击还原 |
| 修补关联 | 调用 AI 分析并补全主线/转折节点的支线关联和伏笔标注 |
| ＋ | 手动添加事件，可选类型和插入位置 |
| － | 进入删除模式，点击节点卡片选中后批量删除 |
| 编辑 | 进入编辑模式，点击节点卡片弹出编辑框修改标题/内容/时间/地点，可变更节点类型 |

**阶段划分（「划分阶段」按钮）：**

在剧情页点击「划分阶段」打开面板：

- **按转折点自动划分** — 每遇到一个转折点就封闭当前阶段，生成若干阶段
- **新建阶段** — 手动创建阶段槽，点击节点将其归入目标阶段（每个节点只属于一个阶段）
- 未分配节点区域提示仍有节点待处理
- 确认划分后节点的 `stageIdx` 字段同步更新，阶段一默认开启

## 角色页

### 角色卡

清洗时自动提取角色，按**主角 / 配角 / 反派 / 其他**分 Tab 展示。每张卡片显示：

- 原始人设（身份 / 外貌 / 性格 / 关系）——来自清洗时 AI 对首次登场描写的提取
- AI 实时人设（如有）——对比当前对话和原著内容更新的动态摘要

**卡片操作：**

| 操作 | 说明 |
|---|---|
| 左侧勾选框 | 开启/关闭该角色的注入（主角默认开启） |
| 眼睛按钮（原始人设） | 控制是否将原始人设注入 prompt |
| 眼睛按钮（AI 实时人设） | 控制是否将 AI 实时人设注入 prompt |
| 编辑 | 展开原始人设和 AI 人设的可编辑文本域，保存后立即生效 |
| 初次登场阶段标签 | 点击跳转到阶段页对应阶段 |

**工具栏：**

- **AI 更新人设** — 对所有已启用角色，结合最近 20 条对话消息和原著剧情节点，调用 AI 生成精简的当前状态摘要；串行执行，受每分钟限速约束
- **按阶段选择** — 下拉面板，按角色初次登场的阶段批量开启/关闭（主角始终跳过）
- **全部开启 / 全部关闭** — 批量操作当前 Tab 的所有角色
- **＋ / －** — 手动添加角色或进入删除模式

---

## 阶段页

划分阶段后，每个阶段显示为一张卡片，包含向量化状态、估算 token 数、阶段标题和概括、节点数量统计。

卡片上的**主线 / 支线 / 转折**数量标签可点击，展开该阶段的内联节点列表，点击任意节点行可直接跳转到剧情页对应节点。

**未向量注入模式（全局切换）：**

| 模式 | 说明 |
|---|---|
| 剧情节点（默认） | 注入该阶段提取的结构化剧情节点文本 |
| 压缩原文 | 注入该阶段清洗压缩后的全文，信息密度更高但 token 消耗更大 |

**开关逻辑：**

| 状态 | 注入行为 |
|---|---|
| 已向量 + 开启 | 发消息时语义召回最相关片段注入 prompt |
| 未向量 + 开启 | 直接将该阶段内容注入 prompt（取决于未向量注入模式设置） |
| 关闭 | 完全跳过，不注入任何内容 |

开启阶段时会自动开启该阶段初次登场的角色，并静默触发一次 AI 实时更新人设。

**演绎提示词（点击代码图标展开）：**  
可编辑注入到酒馆「角色备注」的演绎指导提示词，控制 AI 如何处理用户对原著走向的干预。内置默认提示词包含：

- 用户行为优先于原著时间线的核心原则
- 连锁推演强制执行机制
- 角色内核不变、情境随用户行为动态调整
- 三步时间线处理协议（检索→干涉检测→执行或重写）
- 隐性思维链（每次回复前静默执行）

启用开关控制是否将提示词写入角色备注，关闭时角色备注自动清空。

**批量操作：**

- **全部开启 / 全部关闭** — 批量切换所有阶段的注入开关

**AI 生成：**

- **全部生成** — 对所有阶段调用 AI 生成标题和概括，已有内容也会覆盖
- **补全空白** — 仅生成尚无概括的阶段，已有内容保持不变

## 设置页

### 插件总开关

关闭后停止一切上下文注入，显示「停止注入」提示。处理和向量化功能不受影响。

**全局提示词（点击代码图标展开）：**  
插件每次调用 AI（清洗、单段生成、AI 更新人设、偏差分析、世界设定提取等）时，都会将此提示词前置插入 system 消息最开头。默认内容为内置破限词，可自定义，点击还原按钮恢复默认。

---

### 穿书模式

将剧情节点作为可追踪的任务进度看板，通过状态栏或弹窗显示在酒馆界面，配合 AI 自动注入推进提示词，实现有节奏的剧情推进。

**显示方式（点击设置图标配置）：**

| 选项 | 说明 |
|---|---|
| 状态栏 | 嵌入酒馆底部状态栏，始终可见 |
| 弹窗 | 以悬浮弹窗形式在聊天界面展示 |

两种方式不可同时启用。

**状态栏/弹窗功能：**

- 横向滚动卡片轨道，显示当前阶段的所有剧情节点（主线/支线/关键转折以不同样式区分）
- **节点卡片勾选** — 手动标记某节点已完成，触发推进提示词注入（每对节点只触发一次，防止重复）
- **取消归档** — 点击已完成节点的遮罩层，撤销完成状态
- **阶段切换下拉** — 快速跳转到其他已开启阶段
- **节点列表下拉** — 展示全部节点，分「进行中」和「已归档」两区，支持直接勾选/取消
- **暂停/继续** — 暂停时不自动注入推进提示词，适合自由对话阶段
- **推演按钮** — 调用 AI 基于当前节点和最近对话，生成三条风格各异的后续走向供参考
- **阶段完成提示** — 当前阶段所有节点全部勾选完成后，轨道下方自动显示「本阶段已全部完成」提示条

**穿书模式下的向量召回增强：**  
开启穿书模式后，每次发消息召回向量时，会自动将**当前节点的标题、时间、地点**拼入最新一条消息的 query 头部，作为语义锚点。这样即使对话文本本身没有明确提到场景信息，召回结果也会更贴近当前所处的剧情位置。

**三类提示词（点击代码图标编辑）：**

| 提示词 | 触发时机 | 说明 |
|---|---|---|
| AI 推进提示词 | 勾选节点完成时一次性注入 | 告知 AI 上一阶段已完成，进入下一叙事阶段；包含当前阶段核心、叙事目标、节点处理规则、用户行为优先原则 |
| 持续提示词 | 每条消息均注入 | 穿书模式激活期间持续存在的背景规则 |
| 后文推演提示词 | 点击推演按钮时使用 | 指导 AI 以编剧身份推演三条后续走向 |

三类提示词均支持自定义并可还原默认。

---

### 小说库

保存当前工作区（清洗文本、剧情、角色、阶段划分、向量状态等）为命名快照，支持多本小说切换。新建快照时自动生成唯一标识，「当前」标签始终只跟随最近加载或新建的快照。

| 操作 | 说明 |
|---|---|
| 保存 | 输入快照名称后存入库中 |
| 更新（刷新图标） | 仅当前快照可见，用当前工作区数据覆盖该快照 |
| 重命名（铅笔图标） | 修改快照名称，不影响数据 |
| 加载（下载图标） | 覆盖当前工作区，加载该快照数据，「当前」标签立即刷新 |
| 删除（垃圾桶图标） | 同时清除快照记录和对应的 IndexedDB 向量数据 |

---

### 导入 / 导出

- **导出 ZIP** — 将全部设置和运行时数据（**含**向量数据，以二进制格式打包）、API 配置（**不含** API 密钥）打包为 ZIP 文件，用于跨设备迁移或备份恢复
- **导入 ZIP / JSON** — 支持导入新版 ZIP（含向量）和旧版 JSON（不含向量）；导入内容作为新快照添加到小说库，**不覆盖当前工作区**，导入完成后在小说库中手动加载

> **与旧版差异：** 旧版只能导出 JSON（不含向量），新版 ZIP 导出包含完整向量数据，跨设备恢复后无需重新向量化。

---

### 清除缓存

- **清除向量缓存** — 删除当前小说在 IndexedDB 中的向量数据，不影响剧情/角色/阶段数据
- **清除全部数据** — 二次确认后清空所有工作区数据，完全重置

## 典型使用流程

1. **上传** `.txt` 小说文件
2. 填写清洗 API 配置，点击**开始全自动清洗**
3. 清洗完成后，在**剧情页**检查节点，点击**划分阶段**（或使用按转折点自动划分）
4. （可选）在剧情页点击**世界设定 → AI 全部生成**提取世界观框架
5. （可选）在处理页**文风**卡片中点击**生成文风**提炼写作风格
6. 在**处理页**选择阶段，点击**开始向量化**
7. 在**角色页**检查角色列表，按需开启/关闭，点击**AI 更新人设**生成初始实时摘要
8. 在**阶段页**开启需要参与当前对话的阶段，按需编辑演绎提示词
9. 开始与 AI 对话，每次发送消息前插件自动注入上下文
10. 写到中途可随时点击**分析当前偏差**（剧情页）查看对话与原著的贴合情况
11. （可选）在**设置页**开启**穿书模式**，通过状态栏或弹窗追踪节点进度，使用推演功能参考后续走向
12. 完成后在**设置页 → 小说库**保存快照，方便下次直接加载

## 数据存储

| 数据 | 存储位置 |
|---|---|
| 设置、剧情节点、角色人设、阶段划分、清洗文本、世界设定、文风、穿书进度 | SillyTavern `extension_settings`（随酒馆设置保存） |
| 向量索引 | 浏览器 IndexedDB（`NovelInjectorVectors` 数据库），按 `novelKey` 隔离 |
| 服务端重数据（清洗文本、节点等） | SillyTavern 服务端文件（按 `novelKey` 单独存储，防止浏览器 localStorage 超限） |

不同小说的向量数据通过 `novelKey` 完全隔离，切换小说时不会互相污染。

## 常见问题

**Q：清洗结果里某段一直失败怎么办？**  
A：点击该段的「生成此段」单独重试，或点击「跳过本段」跳过后用「重试失败分段」批量重跑。每段最多自动重试 3 次，若 AI 返回内容缺少结构化元数据（`ni_meta`），插件会先尝试从响应正文中抢救裸 JSON，仍失败才标记为失败。

**Q：向量化后修改了阶段划分，需要重新向量化吗？**  
A：需要。阶段划分变更后，**节点归属发生变化的阶段**（旧阶段和新阶段）向量状态会自动重置为未向量；未涉及变化的阶段不受影响。在处理页重新选择这些阶段向量化即可。

**Q：「剧情节点」注入和「压缩原文」注入有什么区别？**  
A：剧情节点是结构化的事件摘要，token 消耗少、结构清晰；压缩原文是清洗后的流水叙述，信息更完整但 token 消耗较大。可在阶段页的「未向量注入模式」下拉中全局切换。

**Q：导出的 ZIP 包含哪些内容？**  
A：包含所有设置项、运行时数据（清洗结果、剧情节点、角色人设、阶段划分、世界设定、文风、向量完成状态）以及向量数据本身，但**不包含** API 密钥（`cleanKey` / `vecKey`）。旧版导出的 JSON 不含向量，导入后仍需重新向量化。

**Q：导入 ZIP / JSON 会覆盖当前工作区吗？**  
A：不会。导入后作为新快照加入小说库，需要在小说库中手动点击加载才会切换工作区。

**Q：「AI 更新人设」和原始人设有什么区别？**  
A：原始人设来自清洗阶段，记录角色**首次登场时**的固定信息（同名角色只取首次登场段落，后续段不重复录入）。AI 实时人设通过对比当前对话和原著节点动态生成，反映角色**当前状态**。两者注入开关相互独立。

**Q：穿书模式的推进提示词只触发一次是什么意思？**  
A：每对「完成节点 → 下一节点」的组合只会注入一次推进提示词。**但如果对该节点执行「取消归档」，该组合的发送记录会被清除，下次重新勾选完成时将再次触发注入**，以便修正推进提示词。若不希望重复注入，不要对已完成节点执行取消归档操作。

**Q：清洗时 AI 拒绝处理敏感内容怎么办？**  
A：内置提示词已声明任务性质为文学分析，明确要求不得以内容敏感为由拒绝处理。若仍遇到拒绝，建议换用内容审查较宽松的模型（如本地模型）作为清洗 API。

**Q：文风分析采样模式提示「未找到对应段落原文」怎么办？**  
A：文风采样需要原始 `chunks` 数据，该数据仅在当前会话中上传文件后才存在于内存，刷新页面后会丢失。如需在刷新后使用文风功能，请切换到「范文模式」手动粘贴样本，或重新上传文件（无需重新清洗）。
